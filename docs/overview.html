Emergency Guidance Agent
Problem Flow Architecture Demo
Multimodal • Realtime • Guided Action

AI that sees, listens, and coaches users through a high-stress physical task.

Emergency Guidance Agent is a realtime multimodal copilot for lay rescuers. It takes in live audio and video, interprets what the user is showing and saying, tracks workflow state, and responds with the next short actionable instruction.

The problem

In urgent situations, users do not need a long article or generic chatbot answer. They need one grounded next step.

Today

Static instructions break under stress

Search results, videos, and long guides require interpretation at exactly the moment a user is panicked and overloaded.

Gap

Assistants usually do not see the scene

Most systems answer questions in text. They do not track the visible situation or verify whether the user is actually doing the task.

Opportunity

Live multimodal guidance

A system that sees, listens, and stays inside a bounded workflow can provide much more usable help in real time.

Core product loop

The MVP is not broad diagnosis. It is a narrow, high-confidence loop: observe → interpret → instruct → verify → continue.

1

User enters session

Camera and mic are enabled. The system asks the user to show what is happening.

2

Gemini interprets context

Live speech and visual input are used together to understand scene state and user intent.

3

Workflow engine decides step

The application, not the model, decides the current state and the next allowed action.

4

Assistant gives one instruction

The user receives a short spoken and visual instruction in the appropriate language.

5

System observes again

New audio/video context is used to repeat, clarify, or advance to the next step.

What makes this different

This is not a generic chatbot and not a video player. It is a live supervision loop.

Scene-aware

The assistant responds to what is actually visible in front of the user, not just what the user manages to type.

Step-based

The system operates inside a deterministic workflow instead of improvising open-ended advice.

Multilingual

The same realtime loop can support users speaking naturally in different languages.

Architecture

Gemini interprets. The application decides. This separation is the core product and safety architecture decision.

Runtime stack

Frontend client Captures camera and mic, renders instructions, shows transcript, manages Start / Done / Repeat actions.
↓
Realtime transport Streams user audio and video context into the live model session and returns spoken/text responses.
↓
Gemini 3.1 Flash Live Handles multimodal reasoning over speech + vision and generates concise user-facing guidance.
↓
Workflow engine Finite-state machine that determines current step, allowed transitions, repeat logic, and escalation behavior.
↓
Session state Stores scenario, current step, language, uncertainty flags, and user progression for the active session.
Frontend

Owns the experience

  • Camera and microphone capture
  • Session start and teardown
  • Current instruction display
  • Transcript and status UI
  • User controls: done, repeat, exit
Backend

Owns control and state

  • Session manager and active state
  • Workflow finite-state machine
  • Gemini session orchestration
  • Validation of model outputs
  • Safe instruction formatting

Current build scope

We intentionally cut scope to make the loop real, reliable, and demoable.

In scope

Realtime guidance

Live audio/video input, concise spoken response, workflow state tracking, and scene-aware instruction.

Deliberate tradeoff

No video recommendation layer

We removed dynamic tutorial search/playback so the product could focus on live supervision rather than content retrieval.

Out of scope

No open-ended diagnosis

The assistant stays inside a bounded workflow. It is a task copilot, not a general medical expert.

Technology choices

The stack was chosen for fastest path to a convincing live multimodal demo.

Gemini 3.1 Flash Live

Realtime multimodal model for speech + video understanding and live spoken response generation.

Web client

Single-session frontend with camera preview, transcript, and large instruction-first UI for stress-friendly interaction.

State machine backend

Deterministic workflow control so product logic remains consistent even if the model is uncertain.
